# 简单GTO

自用德州扑克 8 人现金局实战训练器。V1 固定规则：8-max，盲注 10/20，200BB 入场，手机竖屏优先，策略表只服务翻前。

## 当前能力

- 8 人现金桌，固定玩家座位，BTN/SB/BB/UTG 等位置按手轮换。
- 完整手牌流程：翻前、翻牌、转牌、河牌、摊牌、弃牌提前结束、全下自动发牌。
- 机器人桌风：顶级强手 + 偏离型真人风格，局后展示个性和数据。
- 翻前范围表：底部 `策略` 或座位旁扑克牌图标打开，只在翻前可用。
- 现金局补码：`选项` 内主动补满；手牌进行中会预约到下一手开始前生效；破产时才强提示。
- 下注器：金额与 BB 双显示，支持快捷尺寸、手动微调、预选动作，`全下` 需要二次确认。
- 抽水：5%，封顶 3BB；No flop, no drop；未跟注筹码不参与抽水。
- 本场总结：机器人数据、本场结果、最近手牌、最大问题。
- 本地保护：启动时自动隔离损坏存档，启动异常时显示恢复入口，不影响继续打牌。

## 本机验证

```sh
bun run verify
```

验证内容包括：

- `bun test` 规则测试。
- 浏览器入口打包检查。

如果 `.tmp-bun-build` 被留下，可以直接删除；它已加入 `.gitignore`。

Windows 旧环境如果仍需要显式注入 `SystemRoot`，可以运行：

```powershell
bun run verify:windows
```

## 本机预览

```sh
bun run serve
```

脚本会输出：

- 本机地址：`http://localhost:4173`
- 同 Wi-Fi 手机可尝试访问的局域网地址。

如果 Windows 当前网络栈报“无法加载或初始化请求的服务提供程序”，这是本机网络服务问题，不代表 App 代码坏了。

## Mac mini 一键打开

```sh
bash scripts/launch-local-mac.sh
```

脚本会确认 4173 端口上运行的是简单GTO；如果服务未启动，会自动启动本地服务，并用默认浏览器打开 `http://127.0.0.1:4173/#autostart` 进入实战。保持终端窗口运行即可持续使用，停止服务按 `Ctrl+C`。

如果 4173 端口被其他程序占用，脚本会直接提示并退出。

## Windows 桌面一键打开

Windows 桌面快捷方式 `简单GTO` 会打开 `scripts/launch-local.ps1`，先确认 4173 端口上运行的是简单GTO，再用浏览器打开 `http://127.0.0.1:4173/#autostart` 并进入实战。

如果上次牌桌存档损坏，页面会自动隔离旧存档并重新进入可启动状态；如果 4173 端口被其他程序占用，启动脚本会弹出明确提示。

## 手机使用路径

最简单试打路径：先在电脑运行 `bun run serve`，手机和电脑连同一个 Wi-Fi，用脚本输出的局域网地址打开。

如果要在安卓上像 App 一样添加到桌面，通常需要 HTTPS 静态托管；本项目是纯静态 Web App，后续可直接放到 Vercel、Cloudflare Pages、Netlify 或 GitHub Pages。
