简单GTO · V2.4 UI 自动化验收 PRD

生成日期：2026-05-13 · 状态：已确权，进入实现

背景

V2.3 已完成 7 个 AI 的翻前、翻后策略和下注尺度优化。当前主要风险不再是单点规则函数，而是用户双击桌面快捷按钮后，真实页面是否稳定进入牌桌、是否白屏、是否有控制台错误、关键面板是否可打开。

之前 UI 验收使用临时 Playwright 脚本完成，无法被后续版本复用。V2.4 需要把这类冒烟验收固化为项目脚本。

目标

  • 提供一条可重复执行的 UI 冒烟测试命令。
  • 自动启动本地服务并访问 #autostart。
  • 覆盖桌面和手机视口。
  • 检查牌桌是否加载、8 个座位是否渲染、选项面板是否可打开。
  • 捕获浏览器控制台 error / warning。
  • 截图保存到系统临时目录，不污染仓库。

验收标准

  • bun run verify 通过。
  • bun run smoke:ui 可在 Mac mini 当前环境运行。
  • smoke 脚本自动启动并停止本地服务。
  • 桌面和手机视口都能加载 简单GTO 牌桌并看到 8 个座位。
  • 桌面视口能打开选项面板，并看到“重新开始”和“重置 AI 对我的记忆”控件。
  • 控制台无 error / warning。
  • 桌面快捷按钮仍指向主项目目录。

使用方式

  • 常规工程验证：bun run verify
  • UI 冒烟验收：bun run smoke:ui
  • 完整本地验收：bun run verify:ui
