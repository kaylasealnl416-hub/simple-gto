# 简单GTO V1 PRD

- 日期: 2026-04-23
- 项目代号: 简单GTO
- 产品定位: 自用、手机优先、安卓可安装的德州扑克现金局实战训练 Web App
- 文档状态: V1 已确权

## 1. 产品目标

本项目不是对外商业化平台，也不是百科式教学站，而是一个为单一用户服务的高压实战训练器。

V1 的目标只有一个:

- 做出一张尽可能接近真实线上现金局的 8 人桌
- 让机器人更像真人强手与真实桌风混合对手
- 让用户能够直接开打，并在赛后快速看懂这一桌都是什么人、自己哪里打偏了

V1 不追求:

- 多模式
- 多桌型
- 多端原生上架
- 复杂设置系统
- 长期数据平台
- 完整翻后 solver 学习平台

## 2. 一句话定义

一个手机竖屏、自用、以真实现金局规则为底、以真人强手感为核心的德州扑克实战训练 Web App。

## 3. 首版范围

### 3.1 游戏类型

- No-Limit Texas Hold'em
- Cash Game
- 8-max

### 3.2 固定桌面配置

- 玩家数: 8 人桌
- 盲注: 10 / 20
- 默认最大带入: 200BB
- 货币展示: 以筹码金额为主，同时显示 BB

展示格式示例:

- `30 / 1.5BB`
- `60 / 3BB`
- `筹码 3860 / 193BB`
- `底池 150 / 7.5BB`

### 3.3 产品形态

- 手机优先 Web App
- 安卓可添加到桌面
- 联网使用
- 不做离线模式
- 不走应用商店上架流程

### 3.4 使用场景

- 个人自用
- 高频直接开打
- 不做账号系统
- 不做云端档案

## 4. 核心用户路径

### 4.1 首页

首页极简，只服务“快速开打”。

保留:

- 主按钮: `开始实战`
- 次入口: `继续上次牌桌`

不保留:

- 功能广场
- 内容流
- 教学课程页入口
- 复杂设置页

### 4.2 牌桌主流程

`开始实战 -> 进入标准 8 人现金桌 -> 打牌 -> 有明显错误才暂停 -> 无问题自动下一手 -> 60 分钟结束或主动结束 -> 赛后查看桌上 bot 个性与数据 -> 查看自己的结果与关键错误`

## 5. 牌桌规则

### 5.1 座位与位置

- 用户视觉座位固定
- 位置按庄位顺序轮换
- 庄位按钮每手顺时针移动
- 用户不手动选座

这意味着:

- 用户屏幕中的位置锚点不变
- 但用户的真实位置身份会在 `UTG / MP / CO / BTN / SB / BB` 等之间轮换

### 5.2 完整手牌流程

V1 必须支持完整现金局街道流程:

- Preflop
- Flop
- Turn
- River
- Showdown

但不强制每一手都打满四街:

- 若翻前全部弃牌，则翻前结束
- 若翻牌下注后其他人弃牌，则翻牌结束
- 只有在真实条件满足时才进入摊牌

### 5.3 补码规则

补码规则参照德州扑克现金局通用规则设计，采用桌注制思路:

- 一手牌进行中不能补码
- 只能在手与手之间补码
- 补码后总筹码不超过该桌最大带入
- 不人为限制固定补码次数
- 非破产状态不自动弹出补码提示；用户通过 `选项` 主动触发补码
- 若用户在手牌进行中触发补码，则预约到下一手开始前生效

说明:

- 这里的“通用规则”以主流现金局常见线上/线下做法为参考
- 本产品不引入“最多补 2 次后结束”的训练器私规

### 5.4 抽水规则

V1 固定:

- 抽水: 5%
- 封顶: 3BB

设计意图:

- 贴近主流线上现金局口径
- 不做可调抽水表

### 5.5 不支持项

V1 明确不支持:

- Straddle 跨注
- Run It Twice

## 6. 计时与预选动作

### 6.1 行动时间

V1 参照主流线上现金局通行做法设计:

- Preflop 未面对加注: 10 秒
- Preflop 面对加注: 15 秒
- Postflop: 15 秒

### 6.2 Time Bank

- 有 Time Bank
- 用于关键决策延长思考
- 场中表现应接近成熟线上现金桌

### 6.3 超时默认动作

- 能过则过
- 面对下注则弃

### 6.4 预选动作

预选动作按线上现金局通用做法设计，保留移动端安全且最常用的一组:

- 弃牌
- 过牌
- 过牌/弃牌
- 跟注
- 跟注任意或等价自动跟注逻辑

原则:

- 不做花哨宏按钮
- 不把牌桌做成答题器

## 7. Bot 桌池设计

## 7.1 总体目标

V1 的真实性优先级:

1. 像真人强手
2. 再像线上现金桌氛围
3. 最后才是装饰性视觉细节

Bot 不是纯 solver 镜像，也不是教学陪练，而是更像真实现金局中的强 regular 与偏离型真人混合桌。

### 7.2 桌池结构

每桌固定:

- 1 名用户
- 7 名 Bot

其中:

- V1 原始口径：3 个顶级强手 Bot + 4 个偏离型 Bot
- V2 覆盖口径：每桌随机 1-3 个 regular，其余为偏离型真人风格 Bot

### 7.3 顶级强手 Bot

顶级强手池采用随机分配，但子类型有限:

- 顶级 TAG
- 顶级平衡派
- 顶级压迫派

共性:

- GTO 基础扎实
- 整体偏强 regular
- 行为更像高水平常规玩家

### 7.4 偏离型 Bot

偏离型用于还原真实现金局的桌风生态。

偏离风格池可包含:

- 紧弱
- 松凶
- 跟注站
- 疯鱼

分配规则:

- 4 个偏离型 Bot 从风格池中抽取
- 同一偏离风格在一桌最多出现 2 人

### 7.5 Bot 风格揭示规则

- 场中不显示 bot 个性
- 场中不显示 HUD
- 整场 session 结束后统一显示 bot 个性与数据

## 8. 赛后页

### 8.1 展示顺序

赛后页按以下顺序展示:

1. 每个 bot 的个性和本场数据
2. 用户本场输赢
3. 用户关键错误总结与回看入口

### 8.2 每个 Bot 的赛后数据

V1 仅显示本场 session 数据，且只保留 4 项:

- 个性标签
- 入池率 VPIP
- 加注率 PFR
- 本场输赢

不做:

- 云端长期画像
- 多场累计档案
- 场中 HUD

V2 覆盖说明：允许本机 localStorage 保存 “AI 对我的记忆”，仅用于 regular 调整与赛后复盘，不引入账号、云端档案或历史 session 列表。

### 8.3 用户结果显示

用户赛后结果采用金额与 BB 双显示:

- 示例: `净输 180 / 9BB`
- 示例: `净赢 420 / 21BB`

## 9. 策略辅助

### 9.1 总原则

V1 牌局引擎是完整现金局流程，但策略辅助只开放 Preflop。

### 9.2 统一范围表面板

V1 只有一个统一的 `Preflop 范围表面板`。

打开入口有两个:

- 用户座位旁的小扑克牌图标
- 底部 `策略` 按钮

它们打开的是同一个面板，不是两套系统。

### 9.3 范围表形态

- 范围表从底部上拉展开
- 仅支持 Preflop
- 不做翻后策略面板
- 不做复杂 solver 解释

### 9.4 范围表布局原则

- 不遮挡用户手牌
- 不遮挡主操作按钮
- 若空间不足，优先内部滚动
- 高度由“手牌与下注区不可遮挡”反推，不写死固定屏占比

### 9.5 范围表显示规则

- 中文固定
- `记分牌 + 百分比` 固定显示
- 纯白牌面
- 花色固定为标准黑红梅方

### 9.6 范围表颜色

- 加注 = 红色
- 跟注 = 蓝色
- 弃牌 = 灰黑色

## 10. Study 模式

V1 保留轻量 `Preflop Study`，但它只是实战辅助层，不是独立产品主角。

允许:

- 查看 8-max 200bb Cash 的 Preflop 范围
- 按位置和行动链切换
- 点击手牌查看动作分布

不允许:

- 翻后 Study
- 任意场景 Solver
- 大量教学文本

## 11. 牌桌交互与下注器

### 11.1 下注器原则

下注器是 V1 核心交互模块，目标是:

- 快
- 准
- 稳
- 不误触
- 不遮挡核心桌面信息

### 11.2 下注器形式

采用:

- 快捷下注按钮为主
- 手动微调为辅

不采用:

- 纯滑杆主导

### 11.3 主操作按钮

根据当前上下文显示:

- Fold
- Check / Call
- Bet / Raise
- All-in

### 11.4 显示格式

场中关键数值统一采用双显示:

- 金额主显示
- BB 辅显示

示例:

- `跟注 20 / 1BB`
- `加注到 60 / 3BB`
- `底池 150 / 7.5BB`

## 12. 暂停与轻复盘

### 12.1 暂停规则

- 无明显问题: 自动进入下一手
- 有明显问题: 暂停并给出轻提示

### 12.2 轻提示格式

V1 暂停提示仅显示:

- 错误发生街道
- 一句中文错误总结

示例:

- `翻前: 跟注范围过宽`
- `翻牌: 持续下注过频`
- `转牌: 跟注过松`

不做:

- 复杂 EV 面板
- 长篇解释
- 多错误排序系统

## 13. Session 规则

### 13.1 结束条件

单场 session 满足任一条件即结束:

- 达到 60 分钟
- 用户主动点击 `结束本场`

### 13.2 手牌历史

V1 仅保留本场 session 内手牌历史:

- 本场内可回看
- 本场结束后清空

不做:

- 多场次历史档案
- 永久保存
- 账号同步

## 14. 视觉与界面风格

### 14.1 风格方向

参考国际主流德扑 App 的交互习惯，但整体视觉走:

- 简洁
- 柔和
- 耐看
- 低疲劳

不走:

- 赌博夜店风
- 霓虹刺眼风
- 电竞重特效风

### 14.2 固定视觉默认值

- 竖屏
- 中文
- 黑灰底
- 柔和深绿桌布
- 大按钮
- 纯白牌面
- 标准黑红花色

### 14.3 底部导航

V1 仅保留:

- `策略`
- `选项`

不保留:

- 表现
- 偏好设置系统
- 多模式导航

### 14.4 选项页

只保留最小工具功能，不承载配置平台职责。

可包含:

- 重新开始
- 返回首页
- 规则说明 / 帮助
- 结束本场

## 15. 明确不做

V1 不做以下内容:

- 账号登录
- 云端训练记录
- 云端长期画像
- 原生 App 上架
- 离线模式
- 复杂偏好设置
- 多桌型
- 多模式线路
- MTT
- SNG
- Short Deck
- 复杂翻后 Study
- 全量 4Bet pot 研究平台
- 全量多人底池研究平台
- 场中 HUD
- Straddle
- Run It Twice

## 16. 技术实现建议

### 16.1 建议架构

- 前端: 手机优先 Web App
- 牌局引擎: 完整 NLHE 状态机
- 策略层: Preflop 范围库
- Bot 层: 行为原型 + 桌池生成器 + 上下文决策
- 赛后层: 本场统计与错误摘要生成器

### 16.2 核心技术难点

V1 最难的不是页面，而是:

- 完整牌局状态机
- 下注合法性与节奏控制
- Bot 行为既像强手又不像脚本
- 8 人竖屏布局下的信息密度控制

## 17. 分阶段开发计划

### 阶段 A: 骨架

- 移动端 Web 外壳
- 首页
- 8 人桌静态 UI
- 基础筹码/盲注/座位模型

### 阶段 B: 引擎

- 发牌与街道推进
- 下注轮转
- 合法动作系统
- 补码、抽水、计时、超时逻辑

### 阶段 C: Bot

- 1-3 regular 随机桌池
- 偏离型风格池
- 单桌随机分配逻辑
- 行动延迟与人味表现

### 阶段 D: 策略辅助

- 统一底部 Preflop 范围表
- 双入口打开逻辑
- Study 轻量学习页

### 阶段 E: 赛后页

- Bot 个性揭示
- VPIP/PFR/输赢统计
- 用户结果页
- 关键错误轻提示

## 18. 验收标准

### 18.1 核心体验验收

- 用户可在安卓手机浏览器中打开并使用
- 可添加到桌面并以 Web App 方式启动
- 首页可以一键进入标准桌
- 牌桌为竖屏可用，核心信息不拥挤
- 手牌、主操作区、范围表三者不互相遮挡

### 18.2 规则验收

- 盲注固定为 10/20
- 初始带入为 200BB
- 补码只能发生在手与手之间
- 非破产补码由用户主动触发，不打断当前手牌
- 抽水固定为 5%，封顶 3BB
- 无 Straddle
- 无 Run It Twice

### 18.3 牌局流程验收

- 每手牌可真实推进至任意街道
- 不强制打满所有街道
- 超时逻辑正确
- 预选动作可正常工作

### 18.4 Bot 验收

- 每桌随机 1-3 个 regular，其余为偏离型 Bot
- 同一偏离型单桌不超过 2 人
- Bot 场中不显示个性
- 赛后可正确显示个性、VPIP、PFR、本场输赢

### 18.5 策略辅助验收

- `策略` 按钮与座位旁图标打开同一底部范围表
- 范围表仅支持 Preflop
- 范围表颜色与显示规则固定
- 范围表不遮挡用户手牌

## 19. 测试方案

### 19.1 规则测试

- 发牌与按钮轮换正确
- SB/BB 扣盲正确
- 翻前行动顺序正确
- 多人弃牌后可在正确街道结束
- 摊牌比较正确

### 19.2 现金局规则测试

- 手牌中不能补码
- 手与手之间可补码
- 抽水百分比与封顶正确
- 超时默认动作正确

### 19.3 UI 测试

- 竖屏主流程完整可操作
- 范围表展开时不盖住用户手牌
- 大按钮区域可稳定点击
- 不同筹码/BB 双显示格式正常

### 19.4 Bot 测试

- 顶级强手池和偏离池分配符合规则
- 偏离型不超过 2 人重复
- Bot 赛后数据统计正确

### 19.5 UAT 建议

- 真实手机上连续打一场 60 分钟 session
- 重点观察:
  - 下注器手感
  - 牌桌节奏是否像真钱线上现金局
  - Bot 是否明显脚本化
  - 范围表是否真的能边打边用
  - 赛后页是否足够帮助理解这一桌人

## 20. 后续版本候选

仅作为后续候选，不进入 V1:

- 复杂翻后策略辅助
- 多模式桌型
- 历史 session 保存
- 账户系统
- 更细的 bot 数据
- 多套抽水规则
- 4Bet / 多人底池研究扩展
